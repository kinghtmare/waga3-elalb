Abstract

This project explores the design and implementation of a highly functional robotic system powered by an ESP32 microcontroller, designed to seamlessly integrate autonomous navigation and manual control capabilities. The robot operates in two distinct modes: Autonomous, where it follows a designated line on the ground using an array of five IR sensors, and Manual, which allows for direct control through a PS4 controller connected via Bluetooth.

The implementation of a PID (Proportional, Integral, and Derivative) control system enables smooth and responsive navigation in Autonomous mode, allowing the robot to adjust its speed and direction based on real-time feedback from the sensors.

To enhance versatility, the robot features four DC motors strategically selected to balance torque and speed, providing optimal performance under varying loads. one servo for the hammr mechanism.

Hardware Components

1. The Microcontroller

ESP32-WROOM-32 Development Board 38-Pin

The ESP32-WROOM-32 Development Board 38-Pin is a powerful and cost-effective microcontroller ideal for IoT projects. It features dual-core processing, Wi-Fi, and Bluetooth/BLE integration, offering versatile connectivity with 38 pins running at 240 MHz

2. Servo Motors

2x L298N Motor Drivers were used to control the four DC motors. The L298N modules provide reliable dual H-bridge motor control, allowing for independent speed and direction management of each motor, which is essential for precise movement and maneuverability of the robot.

2. Servo Motors

MG996R 360 degrees (for the arm) after knowing that a plastic servo would not provide the necessary torque and durability.

3. DC Motors

After calculating the required torque, we determined that four DC motors would be necessary to achieve the optimal speed and torque for our design.

4. Line Tracking Sensor Module 5 Channel

We selected the Line Tracking Sensor Module 5 Channel because it provides five sensors, which is the optimal number for our design. This configuration allows for accurate line detection and effective navigation.

5. Rechargeable Batteries

We have a total of three batteries, providing a combined output of 12V, with an additional spare battery on hand in case of any damage to the others.

Mechanical Design

The Base

We initially selected this base; however, upon further assessment, we discovered that it is insufficient in size to accommodate the required components.

The base wasn't well-balanced, so we were concerned the robot might tip over or end up on its back once we add the arm.

And then we reached to the final base this base represents our most successful design to date. It will be a two-tiered structure, and I would like to detail the placement of each component. The base is exceptionally stable and features a curved section for the infrared (IR) sensors. Additionally, there is a removable line at the back that can be detached if we encounter any difficulties with the length limitation.

Hammer

We initially focused on practicing the mechanics of gears and exploring how to achieve the most stable design for the gripper.

This is the final result we achieved: we utilized a sphere to enable the gripper hand to hold the ball. The design also incorporates two gears that transfer the movement of the servo motor to the gripper's hands.

We added an additional task by creating an arm that holds the gripper. The advantage of incorporating this arm is that, if the ball is dropped, the arm can lower itself to retrieve it.

Software

In our software development, we initially began by implementing manual code and utilizing the Bluetooth module to establish a connection with a smartphone. However, we soon realized that this approach was too slow for our needs. Consequently, we considered the possibility of integrating a PS4 controller to enhance the control and responsiveness of the robot.

The Manual Code (with using the mobile phone)

PS4 Controller Library

The PS4Controller.h library provides an easy way to interface the ESP32 microcontroller with a PlayStation 4 (PS4) controller using Bluetooth. It allows the ESP32 to receive input from the controller, such as button presses, joystick movements, and sensor data, and provides feedback like changing the LED colors on the controller or triggering vibrations.

Library Import
#include <PS4Controller.h>

This line imports the PS4Controller.h library, which contains all the necessary functions to communicate with the PS4 controller over Bluetooth.

Initialization
PS4.begin("40:22:d8:75:20:22");

The PS4.begin() function is used to initialize the Bluetooth communication between the ESP32 and the PS4 controller.

Callback Function
PS4.attach(notify);

The PS4.attach() function allows you to assign a callback function that will be triggered every time the state of the controller changes.

The Final Code

Our final code includes both autonomous mode and manual mode. In autonomous mode, the robot follows a designated line by interpreting the readings from the infrared (IR) sensors. In contrast, manual mode employs user-controlled commands, allowing the operator to navigate the robot between walls and retrieve balls to accumulate points in our basket.
